Customization Guide

This section provides detailed instructions on how to customize all templates in the Elevate package.

Common Customization Options

The following customization options apply to all templates in the package:

Changing Colors and Styles

All templates use CSS variables for easy customization. To change the color scheme:

  1. Open css/variables.css in your code editor
  2. Modify the color variables in the :root section:
:root {
    /* Main Colors */
    --primary-color: #3498db;     /* Change this for primary color */
    --secondary-color: #2c3e50;   /* Change this for secondary color */
    --accent-color: #e74c3c;      /* Change this for accent color */
    --light-color: #ecf0f1;       /* Change this for light color */
    --dark-color: #2c3e50;        /* Change this for dark color */
    
    /* Background Colors */
    --bg-light: #f5f5f5;          /* Change this for light background */
    --bg-white: #ffffff;          /* Change this for white background */
    --bg-dark: #2c3e50;           /* Change this for dark background */
    
    /* Text Colors */
    --text-dark: #333333;         /* Change this for dark text */
    --text-light: #ffffff;        /* Change this for light text */
    --text-muted: #7f8c8d;        /* Change this for muted text */
}

Changing Fonts

To change the fonts used in the templates:

  1. Update the Google Fonts link in the <head> section of index.html:
<link href="https://fonts.googleapis.com/css2?family=YOUR_FONT:wght@300;400;500;600;700&display=swap" rel="stylesheet">
  1. Update the font variables in css/variables.css:
:root {
    /* Fonts */
    --heading-font: 'YOUR_HEADING_FONT', sans-serif;
    --body-font: 'YOUR_BODY_FONT', sans-serif;
}

Modifying Layout Spacing

To adjust spacing throughout the templates:

  1. Modify the spacing variables in css/variables.css:
:root {
    /* Spacing */
    --section-padding: 80px 0;    /* Change this for section padding */
    --container-padding: 0 15px;  /* Change this for container padding */
    --element-margin: 20px;       /* Change this for element margins */
}

Template-Specific Customization

Elevate-Alt: Customizing the Toggleable Sidebar

To customize the sidebar in Elevate-Alt:

  1. Modify the sidebar width in css/variables.css:
:root {
    --sidebar-width: 280px;       /* Change this to adjust sidebar width */
    --sidebar-collapsed-width: 0; /* Width when sidebar is hidden */
}
  1. Customize the sidebar toggle button in css/alternative.css:
.sidebar-toggle {
    position: fixed;
    top: 20px;
    left: 20px;
    z-index: 1000;
    /* Other styles */
}
  1. Adjust the sidebar behavior in js/sidebar.js if needed

Elevate-Colorful: Customizing Gradients and Colors

To customize the vibrant colors in Elevate-Colorful:

  1. Modify the gradient variables in css/colorful.css:
:root {
    --gradient-primary: linear-gradient(135deg, #6a11cb 0%, #2575fc 100%);
    --gradient-secondary: linear-gradient(135deg, #f093fb 0%, #f5576c 100%);
    --gradient-accent: linear-gradient(135deg, #43e97b 0%, #38f9d7 100%);
}
  1. Adjust the color overlay opacity in css/colorful.css:
.color-overlay {
    opacity: 0.8; /* Change this to adjust overlay intensity */
}

Elevate-Portfolio: Customizing Portfolio Grid

To customize the portfolio grid in Elevate-Portfolio:

  1. Modify the grid layout in css/portfolio.css:
.portfolio-grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* Change this to adjust columns */
    gap: 20px; /* Change this to adjust spacing between items */
}
  1. Customize the portfolio filters in index.html:
<div class="portfolio-filter">
    <button class="filter-btn active" data-filter="all">All</button>
    <button class="filter-btn" data-filter="web">Web Design</button>
    <button class="filter-btn" data-filter="graphic">Graphic Design</button>
    <button class="filter-btn" data-filter="photography">Photography</button>
</div>
  1. Make sure portfolio items have the correct data-category attribute:
<div class="portfolio-item" data-category="web">
    <!-- Portfolio item content -->
</div>

Template-Specific JavaScript Features

Elevate-Alt: Sidebar Toggle

The sidebar toggle functionality in Elevate-Alt:

  • Implemented in js/sidebar.js
  • Toggles the sidebar-hidden class on the body element
  • Saves the sidebar state to localStorage
  • Automatically collapses the sidebar on mobile devices
  • Provides a smooth transition when showing/hiding the sidebar

Key functions in sidebar.js:

// Toggle sidebar visibility
function toggleSidebar() {
    document.body.classList.toggle('sidebar-hidden');
    saveSidebarState();
}

// Save sidebar state to localStorage
function saveSidebarState() {
    const isHidden = document.body.classList.contains('sidebar-hidden');
    localStorage.setItem('sidebar-hidden', isHidden);
}

// Load sidebar state from localStorage
function loadSidebarState() {
    const isHidden = localStorage.getItem('sidebar-hidden') === 'true';
    if (isHidden) {
        document.body.classList.add('sidebar-hidden');
    }
}

Elevate-Portfolio: Portfolio Filtering

The portfolio filtering functionality in Elevate-Portfolio:

  • Implemented in js/portfolio.js
  • Filters portfolio items based on their data-category attribute
  • Shows all items when clicking the "All" filter
  • Shows only items matching the selected category for other filters
  • Provides a smooth transition when filtering items

Key functions in portfolio.js:

// Filter portfolio items
function filterPortfolio(category) {
    const items = document.querySelectorAll('.portfolio-item');
    
    items.forEach(item => {
        const itemCategory = item.getAttribute('data-category');
        
        if (category === 'all' || itemCategory === category) {
            item.style.display = 'block';
            setTimeout(() => {
                item.classList.add('show');
            }, 50);
        } else {
            item.classList.remove('show');
            setTimeout(() => {
                item.style.display = 'none';
            }, 300);
        }
    });
}

Responsive Design

All templates in the Elevate package are fully responsive and work well on all devices. This section explains the responsive design features.

Breakpoints

The templates use the following breakpoints:

  • Large screens: 1200px and above
  • Medium screens: 992px to 1199px
  • Tablets: 768px to 991px
  • Mobile: 767px and below

These breakpoints are defined in css/responsive.css:

/* Large screens */
@media (min-width: 1200px) {
    /* Styles for large screens */
}

/* Medium screens */
@media (min-width: 992px) and (max-width: 1199px) {
    /* Styles for medium screens */
}

/* Tablets */
@media (min-width: 768px) and (max-width: 991px) {
    /* Styles for tablets */
}

/* Mobile */
@media (max-width: 767px) {
    /* Styles for mobile devices */
}

Mobile-Specific Features

On mobile devices, the templates include the following features:

  • Adjusted font sizes and spacing for better readability
  • Simplified layouts with single-column content
  • Mobile-optimized navigation
  • Touch-friendly buttons and interactive elements
  • Optimized images for faster loading

Browser Compatibility

All templates in the Elevate package are designed to work well in modern browsers. This section details browser compatibility information.

Supported Browsers

The templates are fully compatible with the following browsers:

  • Google Chrome (latest)
  • Mozilla Firefox (latest)
  • Apple Safari (latest)
  • Microsoft Edge (latest)
  • Opera (latest)

Partial Support

The templates have partial support for:

  • Internet Explorer 11 (basic functionality works, but some visual effects may be limited)
  • Older versions of supported browsers (within the last 2 years)

Known Issues

The following issues may occur in older browsers:

  • CSS variables are not supported in IE11 and below
  • Some CSS Grid and Flexbox features may not work correctly in older browsers
  • CSS animations and transitions may be less smooth in older browsers
  • Some JavaScript ES6 features may not work in older browsers without polyfills

Important: We recommend using a modern browser for the best experience with these templates.

Browser Testing

All templates have been tested in the latest versions of Chrome, Firefox, Safari, and Edge to ensure compatibility. We recommend testing your customized template in multiple browsers before launching your website.
